# Implementation Plan: AgentEscrow (O.A.N)

## Overview

Adapting the original spec to use **React + Rust + PocketFlow** (no Python, Node.js, or Next.js).

## Database Schema

```sql
-- Tasks table
CREATE TABLE tasks (
    id TEXT PRIMARY KEY,
    prompt TEXT NOT NULL,
    bounty_sats INTEGER NOT NULL,
    stake_sats INTEGER DEFAULT 0,
    status TEXT NOT NULL, -- 'pending_payment', 'funded', 'claimed', 'verifying', 'completed', 'failed'
    escrow_invoice TEXT,
    payment_hash TEXT,
    buyer_pubkey TEXT NOT NULL,
    worker_pubkey TEXT,
    worker_invoice TEXT,
    result TEXT,
    verified_at TIMESTAMP,
    created_at TIMESTAMP NOT NULL,
    updated_at TIMESTAMP NOT NULL
);

-- Agents table  
CREATE TABLE agents (
    pubkey TEXT PRIMARY KEY,
    lightning_address TEXT,
    reputation_score REAL DEFAULT 0.0,
    total_tasks INTEGER DEFAULT 0,
    successful_tasks INTEGER DEFAULT 0,
    total_earned_sats INTEGER DEFAULT 0,
    created_at TIMESTAMP NOT NULL
);

-- Payments table (audit trail)
CREATE TABLE payments (
    id TEXT PRIMARY KEY,
    task_id TEXT,
    invoice TEXT NOT NULL,
    payment_hash TEXT,
    amount_sats INTEGER NOT NULL,
    direction TEXT NOT NULL, -- 'in' or 'out'
    status TEXT NOT NULL, -- 'pending', 'paid', 'failed'
    created_at TIMESTAMP NOT NULL,
    settled_at TIMESTAMP
);
```

## API Endpoints

### Public (L402 gated - 1 sat to read task board)

```
GET  /api/tasks              - List available tasks (L402: 1 sat)
GET  /api/tasks/:id          - Get task details (L402: 1 sat)
```

### Task Creation & Escrow

```
POST /api/tasks              - Create task + generate escrow invoice
     Request:  { prompt, bounty_sats, stake_required }
     Response: { task_id, escrow_invoice, payment_hash }

POST /api/webhooks/payment   - Lexe webhook for payment received
     Updates task status: pending_payment -> funded
```

### Task Claiming & Completion

```
POST /api/tasks/:id/claim    - Claim a task (worker stakes if required)
     Request:  { worker_pubkey, worker_invoice }
     Response: { claimed: true } or { stake_invoice }

POST /api/tasks/:id/submit   - Submit task result
     Request:  { result }
     Response: { status: 'verifying' }

GET  /api/tasks/:id/status   - Poll task status
     Response: { status, result?, payout_tx? }
```

### Agent Wallet Integration

```
GET  /api/agent/balance      - Get agent's escrowed balance
POST /api/agent/withdraw     - Withdraw to Lightning address
```

## PocketFlow-Rust Agent Design

### Flow 1: Worker Agent (Claims and completes tasks)

```rust
use pocketflow_rs::{build_flow, Node, ProcessState, Context};

enum WorkerState {
    Idle,
    TaskFound,
    Claiming,
    Working,
    Submitting,
    Completed,
    Failed,
}

// Node: Poll for available tasks
struct PollTasksNode;

// Node: Decide whether to claim based on capabilities
struct EvaluateTaskNode;

// Node: Claim the task (pay stake if needed)
struct ClaimTaskNode;

// Node: Do the actual work using LLM
struct ExecuteWorkNode {
    openai_client: OpenAI,
}

// Node: Submit result to API
struct SubmitResultNode;

let worker_flow = build_flow!(
    start: ("poll", PollTasksNode),
    nodes: [
        ("evaluate", EvaluateTaskNode),
        ("claim", ClaimTaskNode),
        ("work", ExecuteWorkNode::new()),
        ("submit", SubmitResultNode)
    ],
    edges: [
        ("poll", "evaluate", WorkerState::TaskFound),
        ("evaluate", "claim", WorkerState::Claiming),
        ("claim", "work", WorkerState::Working),
        ("work", "submit", WorkerState::Submitting),
        ("submit", "poll", WorkerState::Completed),
        ("submit", "poll", WorkerState::Failed)
    ]
);
```

### Flow 2: Buyer Agent (Creates tasks and receives results)

```rust
enum BuyerState {
    Initializing,
    CreatingTask,
    PayingInvoice,
    AwaitingResult,
    Verifying,
    Completed,
}

struct CreateTaskNode;
struct PayInvoiceNode {
    lexe_wallet: LexeWallet,
};
struct AwaitResultNode;
struct VerifyResultNode;

let buyer_flow = build_flow!(
    start: ("create", CreateTaskNode),
    nodes: [
        ("pay", PayInvoiceNode::new(wallet)),
        ("wait", AwaitResultNode),
        ("verify", VerifyResultNode)
    ],
    edges: [
        ("create", "pay", BuyerState::CreatingTask),
        ("pay", "wait", BuyerState::PayingInvoice),
        ("wait", "verify", BuyerState::AwaitingResult),
        ("verify", "end", BuyerState::Completed)
    ]
);
```

### Flow 3: Verifier Agent (Platform-side verification)

```rust
enum VerifyState {
    Pending,
    CheckingFormat,
    CheckingContent,
    Passed,
    Failed,
}

struct CheckFormatNode;
struct CheckContentNode {
    openai_client: OpenAI, // GPT-4o-mini for cheap verification
};
struct ReleaseEscrowNode {
    lexe_wallet: LexeWallet,
};
struct RefundEscrowNode {
    lexe_wallet: LexeWallet,
};

let verifier_flow = build_flow!(
    start: ("format", CheckFormatNode),
    nodes: [
        ("content", CheckContentNode::new()),
        ("release", ReleaseEscrowNode::new(wallet)),
        ("refund", RefundEscrowNode::new(wallet))
    ],
    edges: [
        ("format", "content", VerifyState::CheckingFormat),
        ("content", "release", VerifyState::Passed),
        ("content", "refund", VerifyState::Failed)
    ]
);
```

## Key Technical Decisions

### Why Lexe instead of MDK?
- **Native Rust SDK**: Lexe has first-class Rust support
- **24/7 Cloud Node**: No node management, always online
- **Self-custodial**: User controls keys via seed phrase
- **L402 Compatible**: Can implement L402 protocol on top

### Why PocketFlow-Rust?
- **Minimal**: ~100 lines, zero bloat
- **Native Rust**: No Python/Node.js needed
- **OpenAI Integration**: Built-in for verification
- **Flow-based**: Perfect for agent orchestration

### Why SQLite?
- **Simplicity**: Single file, no external DB to manage
- **Rust Support**: First-class via sqlx
- **Adequate**: Hackathon scope doesn't need Postgres
- **Easy Demo**: No setup required for judges

## L402 Implementation Strategy

For the Rust backend, we'll implement a custom L402 middleware using Lexe:

```rust
// L402 Middleware for Axum
pub async fn l402_gate(
    amount_sats: u64,
    lexe_wallet: LexeWallet,
    req: Request,
    next: Next,
) -> Result<Response, L402Error> {
    // Check for L402 Authorization header
    if let Some(auth) = req.headers().get("Authorization") {
        let (macaroon, preimage) = parse_l402_header(auth)?;
        if verify_macaroon_and_preimage(&macaroon, &preimage, amount_sats)? {
            return Ok(next.run(req).await);
        }
    }
    
    // Generate invoice and return 402
    let invoice = lexe_wallet.create_invoice(amount_sats, "Task board access").await?;
    let macaroon = generate_macaroon(req.uri().path(), amount_sats)?;
    
    Ok(Response::builder()
        .status(402)
        .header("WWW-Authenticate", format!("L402 macaroon=\"{}\", invoice=\"{}\"", macaroon, invoice))
        .json(L402Challenge { macaroon, invoice, amount_sats }))
}
```
